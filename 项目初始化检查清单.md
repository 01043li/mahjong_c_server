# 麻将游戏项目初始化检查清单

## 第一步：环境准备 ✅

### Ubuntu服务器检查
```bash
# 运行以下命令，记录输出
echo "=== 环境检查 ==="
gcc --version
make --version
git --version
pkg-config --libs json-c 2>/dev/null || echo "json-c未安装"
pkg-config --libs sqlite3 2>/dev/null || echo "sqlite3未安装"
```

**预期输出：**
- gcc版本：9.x 或更高
- make版本：4.x
- git版本：2.x
- json-c和sqlite3应该能正常显示库信息

**如果缺少：**
```bash
sudo apt update
sudo apt install -y build-essential git libjson-c-dev libsqlite3-dev libssl-dev
```

## 第二步：Git配置 ✅

### 1. 配置用户信息
```bash
git config --global user.name "L下生"
git config --global user.email "你的真实邮箱@example.com"
```

### 2. 验证配置
```bash
git config --list | grep -E "user\.(name|email)"
```

**应该显示：**
```

user.email=你的邮箱@example.com
```

## 第三步：创建项目目录 ✅

### 1. 创建目录
```bash

```

### 2. 初始化Git
```bash

```

### 3. 查看状态
```bash

```

**应该显示：**

## 第四步：设置.gitignore ✅

### 1. 创建.gitignore文件
```bash
cat > .gitignore << 'EOF'

*.so
*.a

# 可执行文件

test_runner

# 日志和临时文件

*.tmp

# IDE文件
```

### 2. 第一次提交
```bash
git add .gitignore
```


## 常见问题解决

### 问题1：make编译失败
```bash
# 查看错误信息
make 2>&1 | head -20

# 常见原因和解决：
# 1. 缺少库：sudo apt install libssl-dev
# 2. 权限问题：chmod +x Makefile
# 3. 路径问题：确保在项目根目录
```

### 问题2：git add失败
```bash
# 如果文件太大或有权限问题
git add -f 文件名  # 强制添加
# 或
git add --all     # 添加所有
```

### 问题3：想重新开始
```bash
# 删除项目目录重新开始
cd ~
rm -rf mahjong-game
# 然后从第一步重新开始
```
